Редактирование
===

## Задача
Реализовать CRUD-приложение для управления услугами с возможностью добавления, редактирования и удаления записей.

## Настройка проекта

Создайте новый проект:
```bash
npm create vite@latest editing -- --template react-ts
cd editing
npm install redux react-redux nanoid
npm run dev
```

## Функциональные требования

### Обязательная функциональность

**Добавление услуг:**
- [ ] Форма с полями: название услуги, цена
- [ ] По умолчанию отображается режим добавления
- [ ] Кнопка "Save" добавляет новую услугу в список
- [ ] После добавления форма очищается

![Форма добавления](./assets/add.png)

**Редактирование услуг:**
- [ ] Кнопка "Редактировать" у каждой услуги
- [ ] При клике форма заполняется данными выбранной услуги
- [ ] Кнопка "Save" обновляет существующую запись
- [ ] Кнопка "Cancel" очищает форму и возвращает к режиму добавления

![Форма редактирования](./assets/edit.png)

**Удаление услуг:**
- [ ] Кнопка "✕" у каждой услуги удаляет её из списка
- [ ] Если удаляется редактируемая услуга → форма очищается и возвращается к режиму добавления

### Валидация
- [ ] Название услуги: обязательное поле, минимум 2 символа
- [ ] Цена: обязательное поле, число больше 0
- [ ] Показать ошибки валидации под полями

### UI/UX требования
- [ ] Визуально различать режимы добавления и редактирования
- [ ] В режиме редактирования показать название редактируемой услуги
- [ ] Отображать общее количество услуг

## Архитектура Redux

### Структура состояния
```typescript
interface RootState {
  services: {
    items: Service[];
  };
  form: {
    name: string;
    price: string;
    editingId: string | null; // null = добавление, id = редактирование
    errors: {
      name?: string;
      price?: string;
    };
  };
}

interface Service {
  id: string;
  name: string;
  price: number;
}
```

### Основные Actions
```typescript
// Управление списком услуг
ADD_SERVICE     // { name: string, price: number }
UPDATE_SERVICE  // { id: string, name: string, price: number }
DELETE_SERVICE  // { id: string }

// Управление формой
SET_FORM_FIELD      // { field: 'name' | 'price', value: string }
START_EDITING       // { service: Service }
CANCEL_EDITING      // void
CLEAR_FORM         // void
SET_VALIDATION_ERROR // { field: string, error: string }
```

## Подсказки по реализации

1. **Режимы формы**: используйте `editingId: null` для добавления, `editingId: "123"` для редактирования
2. **Валидация**: создайте отдельную функцию `validateForm(name, price)`
3. **ID генерация**: используйте `nanoid()` для создания уникальных ID
4. **Очистка формы**: не забудьте очищать ошибки валидации
